# Substack下書き生成スキル

音声入力の断片やメモなど、生の素材を受け取り、Substack投稿の下書きを生成する。

## トリガー

- 「Substack下書き:(素材)」
- 「〜のことSubstackに書いて」

## ワークフロー

### Step 1: 核になるエピソードの特定

渡された素材(断片的なメモ・音声入力のテキストなど)から、1つの場面・1つの気づきに絞る。
複数の話が混ざっている場合は、一番具体的で描写できる場面を選ぶ。
素材から読み取れない部分は補わない（Step 3の禁止事項を参照）。

### Step 2: 文体ガイドの参照

`.claude/skills/draft-article/buntai_guide.md` を読み込み、文体を合わせる。
過去記事サンプルが埋まっていない場合は、`memory/`配下や`CLAUDE.md`の発信ライン２の記述を参考にする。

### Step 3: 本文作成

**構成:**
```
■ 書き出し（1〜2行）
   → 具体的な場面・娘の言葉から始める

■ 本文
   → 場面の描写 → 気づき → 静かな結び

■ 締め（1行）
   → 教訓にしない。余白を残す
```

**タイトル案を3つ提示する。**
**本文は1,500〜2,500字。**

**安全ルール（必須・例外なし）:**
- 子どもの特定情報（本名・学校名・地名・通っている施設名など）を書かない
- 素材にないエピソードを捏造しない。描写や心情を大きく脚色しない
- 「〜と思う」「〜かもしれない」など、AIっぽい定型文・一般論の締めを避ける

### Step 4: 保存

`drafts/YYYY-MM-DD_[タイトル].md` として保存する。
保存後、次の一言を添える:
```
[ファイル名] に保存しました。
```

## 品質チェック（出力前に確認）

- [ ] 子どもを特定できる情報が入っていないか
- [ ] 素材にない出来事を付け足していないか
- [ ] 締めが教訓・励ましになっていないか
- [ ] CLAUDE.mdの発信ライン２のトーンと合っているか
